SDK and Sample Application Delivery

PenguinIN shares the Mobile SDK as a .framework file. Then, this file is dragged and dropped into an iOS project's Frameworks, Libraries, and Embedded Content section. After that, in the Embed config field, check the "Embed and Sign" option. Compatibility check must performed to ensure that that SDK has been built with the same XCODE version as the target.

For the sample application, project is already pre-configured and ready for build. To Download the SDK and Sample app Click here.

Supported minimum iOS version: 12.0



Sample Application

The sample application has two modes of operation:

  • Simulation mode (for more info about simulation Click here. )
  • Live mode
In simulation mode , the application runs a pre-configured scenario with predetermined list of positions. This is the default configuration for the sample app. In live mode, the position is fetched from the position engine service and of course requires complete BLE beacon roll-out and site calibration data to have been completed. In both modes, connectivity to the position engine service and the presence of at least one BLE beacon is required.

At this stage one should be able to see the map loaded with a blue dot. A single hardcoded POI (shown as a grey square highlighted in red) has been defined to demonstrate the navigation example. Simply click on the highlighted POI zone on the map and click on start navigation.


Now that we have experienced the features of the sample app, we will now explain its inner workings.

The sample app is intended to show a full wayfinding journey, present the two wayfinding modes available and showcase common API usage.

PenNav has two wayfinding modes, namely: free-style and navigation. In free-style the user does not specify a destination. The position returned from the SDK reflects the current position of the user. In navigation mode, the user specifies a destination POI. The API offers a shortest path route from the current position to the POI entrance location. This route is returned as a list of edges. In this mode, an up-to-date position along the navigation path is returned by the SDK. The map image, the route and the location along the route must be drawn by the app developer according to the intended UI rendering guidelines. If the user deviates from the pre-calculated route, the SDK returns a wrong-path error. The app developer may elect to reflect that  in the app space for better navigation experience.

In a multi-floor and/or multi-venue navigation scenario, the SDK continuously returns the venue (i.e. building) and the floor which shall be used by the app developer to update the currently displayed map image.

The app developer may also call the API to list all available POIs to create – for example – a directory listing inside the app. Furthermore, the app developer shall use the API to retrieve the map image for a particular floor. They can also use the API to return the zone (defined as a polygon) which defines the boundaries of a particular POI on the map image. This zone definition can be leveraged in creating gesture interactions with the map image. This feature is implementation specific.

The above is only a brief listing of the capabilities of the API. For a comprehensive list, the app-developer is referred to the API documentation.

SDK: Methods and Integration

In this section we present the methods availed to the application developer through our SDK followed by best practices to integrate it within a mobile application.


4.1 initialize

Returns initialization success or failure as Boolean value.

Operation

These steps describe the process of initialization:

  • Validate OS support and server connectivity.
  • Initialize input sources for positioning platform.
  • Detect current venue and get venue data (will use data API): * Venue specific positioning information * Floors details * Edges details
  • Perform device specific calibration
  • Establish connectivity requirements.
  • Save the results and Log for analytics (if supported)

Parameters

Parameters Description Value Type
userID Front End UserID String
ipAddressDetails Penguin Data API IP address (mandatory)
Penguin Position API IP address (mandatory)
Third party API IP address (optional) List(String)
noneIMU Enable Just JAR None IMU calculation Boolean
regionUUIDs Mounted region/beacons UUID for scanning BLE List(String)

Response

Value Description Value Type
Result Success: 1 or Fail: 0

Error Code. Warning Code. separated by “_” ex: “10_0” or “0_12” | String |

4.8 setMode

Set the position mode and scanning time frequency:

  • Background: * Low frequency position scanning for logging and analytics.
  • Freestyle: * High frequency position scanning for trusted position on the venue. This mode assumes that the user is not wayfinding between a known source and a destination POI.
  • Navigation: * High frequency position scanning for trusted position while navigating to a known destination with a know route already identified.

Parameters

Parameters Description Value Type
ModeID BackGround,FreeStyle,Navigation Int
TimePeriod Time Period in ms Int

Response

Value Description Value Type
Success Updated the mode successfully Boolean

Modes

ID Description Scan Time / milliseconds
1 Background (Default) 10000
2 Free Style 2000
3 Navigation 2000


4.9 getMyPosition

Returns last detected position concatenated together with "_" separator:

  • Current Floor ID.
  • Position point (X, Y).
  • Accuracy circle radius in pixels.
  • Current Edge ID.

Recommendation:

This method provides an update on the position of the mobile application user. It is not recommended to call it with high frequency. The recommended value is 500msec-2000msec and should be adjusted according to the desired UI behavior.

Operation

  • Concatenate FloorID, X, Y, Accuracy Circle Radius , Current Edge ID together with "_" separator:
    • Ex:
      • Current Floor ID: 2.
      • My Position (X, Y): (3000, 5100).
      • Accuracy radius in pixels: 120.
      • Current Edge ID: 150.
      • Result:
        • 2_3000_5100_120_150

Parameters

N/A

Response

Value Description Value Type
Position FloorID, X, Y, Accuracy Circle Radius , Current Edge ID
Concatenated together with "_" separator 		String

Operation

  • Returns the projected point (X,Y)

Parameters

Value Description Value Type
